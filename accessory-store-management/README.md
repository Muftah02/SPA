# نظام إدارة محل الإكسسوارات

نظام إدارة متكامل لمحلات الإكسسوارات مبني بـ Laravel 12 و Livewire، مصمم للعمل محلياً (Offline) بدون الحاجة لإنترنت أو استضافة خارجية.

## المميزات الرئيسية

### 🏪 إدارة المحل
- لوحة تحكم شاملة مع إحصائيات فورية
- إدارة المنتجات والتصنيفات
- إدارة المخزون مع تنبيهات النقص
- إدارة الموردين والعملاء

### 💰 نقطة البيع (POS)
- واجهة بيع سريعة وسهلة الاستخدام
- إنشاء فواتير فورية
- دعم طرق دفع متعددة (نقد، شبكة، آجل)
- طباعة الفواتير

### 📊 التقارير والتحليل
- تقارير المبيعات اليومية والشهرية
- تقارير الأرباح والخسائر
- تقارير المخزون ونقص البضائع
- تقارير ديون العملاء والموردين

### 🌍 دعم اللغة العربية
- واجهة عربية بالكامل
- دعم RTL (من اليمين لليسار)
- رسائل التحقق والأخطاء بالعربية
- تقارير وفواتير عربية

## المتطلبات التقنية

### متطلبات الخادم
- PHP 8.4 أو أحدث
- MySQL 8.0 أو أحدث
- Composer
- Node.js و npm (للتطوير)

### المكتبات المستخدمة
- **Laravel 12**: إطار العمل الأساسي
- **Livewire 3**: لإنشاء واجهات تفاعلية
- **Tailwind CSS**: لتصميم الواجهات
- **SQLite**: قاعدة البيانات المحلية

## التثبيت والإعداد

### 1. استنساخ المشروع
```bash
git clone <repository-url>
cd accessory-store-management
```

### 2. تثبيت التبعيات
```bash
# تثبيت تبعيات PHP
composer install

# تثبيت تبعيات Node.js
npm install
```

### 3. إعداد البيئة
```bash
# نسخ ملف البيئة
cp .env.example .env

# توليد مفتاح التطبيق
php artisan key:generate
```

### 4. إعداد قاعدة البيانات
```bash
# تشغيل الهجرات
php artisan migrate

# إدراج البيانات التجريبية
php artisan db:seed
```

### 5. بناء الأصول
```bash
# بناء أصول CSS و JavaScript
npm run build
```

### 6. تشغيل التطبيق
```bash
php artisan serve
```

سيكون التطبيق متاحاً على `http://localhost:8000`

## بنية المشروع

### النماذج (Models)
- `User` - المستخدمون
- `Category` - تصنيفات المنتجات
- `Product` - المنتجات
- `Supplier` - الموردون
- `Customer` - العملاء
- `Sale` / `SaleItem` - المبيعات وعناصرها
- `Purchase` / `PurchaseItem` - المشتريات وعناصرها
- `Payment` - المدفوعات
- `Setting` - إعدادات النظام

### مكونات Livewire
- `Dashboard` - لوحة التحكم
- `ProductManager` - إدارة المنتجات
- `CategoryManager` - إدارة التصنيفات
- `SupplierManager` - إدارة الموردين
- `CustomerManager` - إدارة العملاء
- `SaleManager` - إدارة المبيعات
- `PurchaseManager` - إدارة المشتريات
- `PointOfSale` - نقطة البيع
- `ReportDashboard` - التقارير

## قاعدة البيانات

### جداول رئيسية
- `users` - المستخدمون
- `categories` - التصنيفات
- `products` - المنتجات
- `suppliers` - الموردون
- `customers` - العملاء
- `sales` / `sale_items` - المبيعات
- `purchases` / `purchase_items` - المشتريات
- `payments` - المدفوعات
- `settings` - الإعدادات

### العلاقات
- منتج ينتمي لتصنيف واحد
- عملية بيع لها عدة عناصر
- عملية شراء مرتبطة بمورد واحد
- المدفوعات مرتبطة بالمبيعات/المشتريات (Polymorphic)

   ```bash

   ```

2. **مشاكل في الأصول**
   ```bash
   npm run build
   ```

3. **مشاكل في التخزين المؤقت**
   ```bash
   php artisan cache:clear
   php artisan config:clear
   php artisan view:clear
   ```
